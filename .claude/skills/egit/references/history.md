# 作業履歴（会話をまたいで作業に戻るため）

作業場所を作成・切替・セーブするたびに `.git/egit-history.md` を更新する。
これにより、新しい会話でも「さっきの作業に戻りたい」に対応できる。

## ファイル形式

`.git/egit-history.md`（gitには追跡されない、プロジェクトごとに1つ）:

```markdown
# 作業履歴

| # | 作業場所 | 状態 | 最終更新 | やっていたこと |
|---|---------|------|---------|--------------|
| 1 | feature/proposal | 作業中 | 2026-04-03 | 提案書の第1章を作成中 |
| 2 | feature/fix-header | 合流済 | 2026-04-02 | ヘッダーのレイアウト崩れを修正 |
| 3 | feature/estimate | 一時保管 | 2026-04-01 | 見積書を途中まで作成 |
```

## 更新ルール

- **作業場所を作成したとき** → 新しい行を先頭に追加（状態: 作業中）
- **セーブしたとき** → 該当行の「やっていたこと」をセーブ内容から要約更新、最終更新を更新
- **合流したとき** → 状態を「合流済」に変更
- **一時保管したとき** → 状態を「一時保管」に変更
- **最大10件** → 11件目が入ったら最も古い「合流済」を削除。合流済がなければ最も古い行を削除
- 「やっていたこと」は **150文字以内** で、ユーザーが見てすぐ思い出せる内容にする
- main は履歴に記録しない

## 「さっきの作業に戻りたい」への対応

**トリガー**: 「さっきの作業に戻りたい」「前の作業の続き」「途中だったやつ」等の発言

手順:
1. `.git/egit-history.md` を読む（なければ `git branch` の一覧から作業場所を列挙）
2. 「作業中」または「一時保管」の作業場所を提案:
   ```
   前回の作業が残っています。どれに戻りますか？

   1. 「提案書の第1章作成」の続き（feature/proposal）
   2. 「見積書」の続き（feature/estimate・一時保管あり）
   3. やっぱりやめる
   ```
3. ユーザーが選択したら:
   a. 未セーブの変更がある場合 → 先にセーブ or 一時保管を提案（「新しい作業場所をつくる」と同じ流れ）
   b. 選択した作業場所に切り替え（`git checkout`）
   c. 一時保管がある場合 → `git stash pop` で復元し報告
   d. 報告: 「○○の作業に戻りました！ 続きからどうぞ」
