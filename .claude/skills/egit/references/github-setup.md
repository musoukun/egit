# GitHub連携セットアップ手順

環境構築で「GitHubでも管理」が選ばれた場合の詳細手順。

## ステップ1: ghコマンドの確認

a. `gh --version` でghがインストールされているか確認
b. **ghが入っていない場合** → インストールを提案:
   ```
   GitHubと連携するには「gh」というツールが必要です。
   自動でインストールしますか？

   1. インストールする
   2. やっぱりこのPCだけで管理する
   ```
   - 選択肢1 → `winget install --id GitHub.cli` を実行
   - インストール完了後:
     ```
     インストールが完了しました！
     ただし、このツールを使えるようにするために、PowerShellを一度閉じて、新しいウィンドウで開き直す必要があります。

     ※新しいタブではなく、新しいウィンドウで開いてください。
     開き直したら、もう一度ここに来て「GitHubで管理したい」と言ってください。
     ```
     → ここで一旦終了。ユーザーがPowerShellを再起動して戻ってくるのを待つ

## ステップ2: GitHubへのログイン

c. `gh auth status` で認証チェック
d. **未認証の場合** → ログインを案内:
   ```
   GitHubにログインする必要があります。
   今からブラウザが開くので、画面の指示に従ってログインしてください。

   準備ができたら「OK」と言ってください。
   ```
   - ユーザーが「OK」→ `gh auth login --web -p https` を実行
   - ターミナルに表示されるワンタイムコードを案内:
     ```
     ブラウザが開きました。
     画面に表示されたコード（XXXX-XXXX）をブラウザに入力して、「Authorize」ボタンを押してください。
     終わったら「できた」と言ってください。
     ```
   - ユーザーが「できた」→ `gh auth status` で認証成功を確認
   - 失敗していたら再案内

## ステップ3: リポジトリ作成と初回セーブ

e. `git init`
f. デフォルトブランチが `master` の場合は `git branch -m master main` で `main` にリネーム
g. `.gitignore` 作成
h. プロジェクト名からリポジトリ名を提案し確認
i. 公開/非公開を確認:
   ```
   このプロジェクトの公開設定はどうしますか？
   1. 非公開（自分だけ見える）← おすすめ
   2. 公開（誰でも見える）
   ```
j. `gh repo create` でリポジトリ作成
k. 全ファイルを初回セーブ＆アップロード
l. 報告: 「環境をつくりました！ GitHubにもバックアップされます」
