# egit スキル

## TL;DR

プログラミングの作業中、ファイルを壊してしまったり、前の状態に戻したくなることがあります。
この仕組みは、AI（CLIツール）に話しかけるだけでファイルの変更履歴を自動で残してくれます。
難しい操作は一切不要です。

- 一連のテストをしています。難しい言葉は注釈をつけて、イメージしやすい言葉に置き換えています。
- **ユーザがコマンドを打つような形式ではなくてAIから提案するようにしています。**
- ですがが使い勝手はまだ、極められてません。。。**どうか皆さんでもっといいスキルに変更してほしいです。**

## この仕組みを作った背景

時間をかけて作ったファイルが、ちょっとした操作ミスで消えてしまう。
誰しも経験があると思います。

ファイルの変更履歴を残す仕組み（Git）は昔からありますが、使うにはコマンドや概念の理解が必要で、ハードルが高いのが現実です。
この仕組みは、そのハードルをゼロにするために作りました。

AI（CLIツール）に話しかけるだけで、**裏側でGitが動いて変更履歴を記録してくれます。**
あなたがGitを知っている必要はありません。

開発向けだけではなく、それ以外の作業でも使えるようにSkillsを考えてみました。

## やりたいこと別の使い方

### 変更履歴を残せるようにする

新しいプロジェクトを始めるとき、AI（CLIツール）に「変更履歴を残したい」と伝えるだけで準備が整います。

あなた:
```
ここでWebサイトを作りたいんだけど、ファイルの変更履歴を残せるようにしたい
```

AI（CLIツール）:
```
このプロジェクトにはまだバージョン管理がありません。
ファイルの変更履歴を残せるようにしますか？

1. このPCだけで管理 - 変更履歴をこのPCに保存します
2. GitHubでも管理 - クラウドにバックアップされ、他の人とも共有できます
```

番号を選ぶだけで完了です。

あなた:
```
1
```

AI（CLIツール）:
```
環境をつくりました！ これからファイルの変更履歴が記録されます。

→ このまま作業を始めてOKです。キリのいいところで「セーブして」と言ってください。
```

これで準備完了です。あとは普通に作業を進めてください。

---

### ファイルの状態をセーブする

ゲームのセーブと同じ感覚です。キリのいいところで「セーブして」と伝えてください。

あなた:
```
セーブして
```

AI（CLIツール）:
```
セーブしました！（3個のファイル）
「追加: ログイン画面のUIを実装」

次にできること:
  → このまま作業を続けてOK
  → 作業が完了したなら「メインに合流して」と言ってください
```

セーブしておけば、あとで「あの時の状態に戻したい」が可能になります。

※セーブを忘れていても、変更が溜まってくるとAI（CLIツール）が「セーブしますか？」と声をかけてくれます。

---

### 別の作業を始めたい

今の作業を中断して、別の機能を作りたくなったとき。

あなた:
```
ちょっと別の作業したいんだけど、お問い合わせページを作りたい
```

AI（CLIツール）はまず、今の作業の状態を確認してくれます。

保存していない変更がある場合:
```
今の作業場所に保存していない変更があります。

1. 変更をセーブしてから移動する
2. 変更を一時保管して移動する（あとで戻せます）
3. やっぱりやめる
```

番号を選ぶだけで、今の作業を安全に退避してから新しい作業に移れます。

---

### 前の作業に戻りたい

別の作業をしていて、前の作業に戻りたくなったとき。

あなた:
```
さっきの作業に戻りたい
```

AI（CLIツール）:
```
前回の作業が残っています。どれに戻りますか？

1. 「ログイン画面のUI実装」の続き
2. 「お問い合わせフォーム」の続き（一時保管あり）
3. やっぱりやめる
```

一時保管していたファイルも自動で復元されます。

---

### 作業をメインに反映する

作業が完了したら、メインの保管場所に合流させます。

「メインの保管場所」とは、完成したファイルを置いておく場所です。作業中の変更は別の場所で進めて、完成したらメインに持っていく、という流れになっています。

あなた:
```
この作業おわったからメインに反映して
```

AI（CLIツール）:
```
メインに合流しました！

次にできること:
  → 新しい作業を始めるなら「新しい作業を始めたい」と言ってください
  → このまま続けるならメインの保管場所で作業できます
```

---

### 同じ箇所を別々に変更してしまったとき

複数の作業で同じファイルの同じ行を変更すると、どちらを採用するか聞かれます。

```
同じ箇所が別々に変更されていました！

ファイル: config.js（1行目）
  合流先にあった方: color = "green"
  あなたの変更:     color = "red"

どちらを残しますか？
  1. あなたの変更を使う（"red"）
  2. 合流先にあった方を使う（"green"）
  3. 両方残す
  4. 別の内容に書き直す
```

番号を選ぶだけで解決できます。ファイルが壊れることはありません。

---

### 特定のファイルを管理対象から外す

「このファイルはWeb上にあげたくない」「セーブの対象にしたくない」というファイルがある場合、無視リストに追加できます。

あなた:
```
logsフォルダを管理対象から外して
```

AI（CLIツール）:
```
logsフォルダを無視リストに追加しました。
今後このフォルダはセーブの対象になりません。
```

管理対象から外しても、ファイル自体は削除されません。セーブや共有の対象から外れるだけです。

こんなファイルを外しておくと安心です。

| 種類 | 例 | 理由 |
|------|-----|------|
| 機密情報 | `.env`、パスワードが書かれたファイル | Web上に漏れるのを防ぐ |
| 大きいファイル | 動画、ZIPファイル | 容量が大きすぎて管理に向かない |
| 自動生成されるファイル | `node_modules/`、`dist/` | いつでも復元できるので管理不要 |
| ログ | `logs/`、`*.log` | 一時的なデータなので管理不要 |

※パスワードや秘密情報を含むファイル（`.env` など）は、環境構築時に自動で無視リストに追加されます。

---

### 今の状態を確認する

いつでも「今どうなってる？」と聞けます。

あなた:
```
今ってどういう状態？
```

AI（CLIツール）:
```
今の状態:
  場所: feature/add-login（メインではありません）
  未セーブの変更: 3個のファイル
  メインと比べて: 2回分の変更があります
  最後のセーブ: 「追加: ログインフォームを実装」（15分前）

できること:
  → 「セーブして」で変更を保存
  → 「メインに合流して」で作業を完了
```

---

## 覚えておくこと

この仕組みを使うために覚えることは3つだけです。

| やりたいこと | 言うこと |
|------------|---------|
| 変更を保存したい | 「セーブして」 |
| 別の作業をしたい | 「別の作業をしたい」 |
| 完成したものを反映したい | 「メインに合流して」 |

あとはAI（CLIツール）が案内してくれるので、選択肢の番号を選ぶだけで進められます。

※パスワードや秘密情報が書かれたファイルは、安全のため自動的にセーブ対象から除外されます。
