# egit テストシナリオ集

## 使い方

1. 新しいターミナルで **別のフォルダ** を作って `claude` を起動
2. 各シナリオのプロンプトを順番にコピペして送信
3. チェック項目を確認

```powershell
# テスト用フォルダ作成（PowerShell）
mkdir D:\tmp\egit-test
cd D:\tmp\egit-test
claude
```

---

## シナリオA: 初めてのプロジェクト（環境構築→作業→セーブ）

### A-1: 環境構築
プロンプト:
```
ここでWebサイトを作りたいんだけど、ファイルの変更履歴を残せるようにしたい
```

チェック項目:
- [ ] 「バージョン管理がありません。つくりますか？」的な提案が出るか
- [ ] 選択肢が番号で選べるか（1.このPC / 2.GitHub）
- [ ] `/egit init` とかコマンドを打てとは言わないか
- [ ] 「はい」や「1」で進められるか

### A-2: 環境構築への返答
プロンプト:
```
1
```

チェック項目:
- [ ] ローカルで環境が作られるか
- [ ] .gitignoreが作られるか
- [ ] 「次にやること」が提案されるか
- [ ] 提案が平易な日本語か

### A-3: ファイルを作る
プロンプト:
```
index.htmlを作って。「Hello World」って表示するやつ
```

チェック項目:
- [ ] ファイルが作られるか
- [ ] （変更が溜まっていれば）セーブの提案が出るか

### A-4: セーブする
プロンプト:
```
セーブして
```

チェック項目:
- [ ] mainにいる場合、作業場所が自動で作られるか
- [ ] 作業場所を作った理由が説明されるか
- [ ] セーブメッセージが日本語で自動生成されるか
- [ ] 「次にできること」が提案されるか
- [ ] `/egit save` とかのコマンドは出てこないか

---

## シナリオB: 作業場所の切り替え（別作業に移りたい）

### 前提
シナリオAが完了した状態で続ける。

### B-1: 途中で別の作業をしたくなる
プロンプト:
```
style.cssを追加して。bodyの背景を水色にするやつ
```
（セーブせずにそのまま↓）

### B-2: 未セーブのまま別作業を始めたい
プロンプト:
```
ちょっと別の作業したいんだけど、お問い合わせページを作りたい
```

チェック項目:
- [ ] 「保存していない変更がある」と気づいてくれるか
- [ ] 選択肢が出るか（セーブしてから移動 / 一時保管して移動 / やめる）
- [ ] 「ブランチ」「stash」等のGit用語が出ていないか
- [ ] 番号を選ぶだけで進めるか

### B-3: 一時保管を選択
プロンプト:
```
2
```

チェック項目:
- [ ] 一時保管が実行されるか
- [ ] 新しい作業場所が作られるか
- [ ] 「あとで戻せます」的な安心感のある案内があるか

### B-4: 元の作業に戻りたい
プロンプト:
```
さっきの作業に戻りたい
```

チェック項目:
- [ ] 元の作業場所に戻れるか
- [ ] 一時保管していたファイルが復元されるか
- [ ] 復元されたことが伝わるか

---

## シナリオC: メインに合流

### 前提
シナリオAまたはBで作業場所にセーブ済みの状態。

### C-1: 合流する
プロンプト:
```
この作業おわったからメインに反映して
```

チェック項目:
- [ ] 未セーブの変更があれば先にセーブしてくれるか
- [ ] メインに合流が実行されるか
- [ ] 「合流しました」的な報告が出るか
- [ ] 「merge」「branch」等のGit用語が出ていないか
- [ ] 「次にできること」が提案されるか

---

## シナリオD: コンフリクト（同じ箇所の競合）

### 準備（手動でやる）
```
これからコンフリクトのテストをしたい。以下を順番にやって。
1. mainでconfig.jsを作って。中身は `const color = "blue";`
2. セーブして
3. 新しい作業場所をつくって、config.jsのcolorを"red"に変えてセーブ
4. メインに戻ってconfig.jsのcolorを"green"に変えてセーブ
5. さっきの作業場所をメインに合流させて
```

チェック項目:
- [ ] 競合が検出されるか
- [ ] 「同じ箇所が変更されていました」的な説明が出るか
- [ ] 選択肢が番号で出るか（1.あなた側 / 2.メイン側 / 3.両方 / 4.自分で書き直す）
- [ ] 「conflict」等のGit用語が出ていないか

### D-2: 競合を解消
プロンプト:
```
1
```

チェック項目:
- [ ] 選んだ側の変更が採用されるか
- [ ] 合流が完了するか

---

## シナリオE: 変更の蓄積→リマインド

### E-1: たくさんファイルを作る
プロンプト:
```
page1.html から page6.html まで6個のHTMLファイルを作って。中身は各ページ番号を表示するだけでいい
```

チェック項目:
- [ ] 回答の最後に「変更が溜まっています」的なリマインドが出るか
- [ ] 「セーブして」と言えばいいと案内されるか（`/egit` ではなく）

### E-2: もっと溜める
プロンプト:
```
page7.html から page12.html まで追加して
```

チェック項目:
- [ ] より強いリマインドが出るか（「かなり溜まっています！」的な）

---

## シナリオF: 新しい会話で「前の作業に戻りたい」

### 前提
シナリオA〜Cをやった後、一度 `/exit` してclaudeを再起動する。
```powershell
cd D:\tmp\egit-test
claude
```

### F-1: 前の作業に戻りたい
プロンプト:
```
さっきの作業に戻りたい
```

チェック項目:
- [ ] 作業履歴から候補が表示されるか
- [ ] 「〜の作業ですか？」的に聞いてくれるか
- [ ] 番号を選ぶだけで戻れるか
- [ ] 一時保管していたファイルがあれば復元されるか
- [ ] 「branch」「stash」等のGit用語が出ていないか

### F-2: 作業履歴の確認
プロンプト:
```
今まで何の作業したっけ？
```

チェック項目:
- [ ] 作業履歴の一覧が表示されるか
- [ ] 各作業が何をしていたか分かる説明がついているか
- [ ] 「作業中」「合流済」「一時保管」の状態が分かるか

---

## シナリオG: 状態確認

### G-1: 今の状態を聞く
プロンプト:
```
今ってどういう状態？
```

チェック項目:
- [ ] 作業場所の名前が表示されるか
- [ ] 未セーブの変更数が表示されるか
- [ ] 「できること」の提案が出るか
- [ ] Git用語なしで理解できるか

---

## シナリオH: GitHub連携（push含む）

### 準備
新しいフォルダで始める。
```powershell
mkdir D:\tmp\egit-test-github
cd D:\tmp\egit-test-github
claude
```

### H-1: GitHub付きで環境構築
プロンプト:
```
ここで新しいプロジェクト始めたい。GitHubにもバックアップしたい
```

チェック項目:
- [ ] GitHub連携の選択肢が出るか
- [ ] 認証チェックが行われるか
- [ ] リポジトリ名の確認があるか
- [ ] 公開/非公開の確認があるか
- [ ] push（アップロード）が成功するか

---

## 全体チェック（全シナリオ共通）

- [ ] `/egit` などのコマンドをユーザーに打たせる場面がないか
- [ ] 「はい」「番号」だけで全操作が完了するか
- [ ] 毎回「次にできること」が提案されるか
- [ ] Git用語が裸で使われていないか（初回は（）補足OK、2回目以降は平易語のみ）
- [ ] エラーが起きても「何をすればいいか」が明確か
